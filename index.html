Berkeley Protest Mapping

(Work in Progress)


This interactive project is currently a work in progress, started during my time at the Daily Californian for Overflow, the data journalism side of the newspaper. The goal was two-fold: (1) to be able to map all the protests that happen near UC Berkeley and see if they follow similar paths, and (2) to assist Daily Cal reporters in their coverage. Look out for this project to be updated.

This project was inspired by the following tweet, made during Black Lives Matter protests of December 2014.


San Pablo and 40th pic.twitter.com/hTnDungOc6

— Sophie Ho (@sophanho) December 10, 2014

For the first goal I envisioned a layered heat map spread across the city, and to do this I needed a dataset with the coordinates of the path of the protest. Unfortunately, there wasn't a great way to get historical data, so I'd have to settle for future protests (which I'm sure will exist). I also wanted to aid reporters, so that the protest could be seen moving, instead of just the intersections they passed. So this is why I'm using Django, to get real-time tweets, and to have a database.

Below is a screenshot from the Heroku app (the live version is currently available here). The current points are all dummy tweets that I created while walking to test the app.

The app works, in the sense that if you post a tweet, it will show up. It has yet to be tested in an actual protest though. I have not yet started on the layering of multiple protests, either. Right now the tweets are not bounded by location or user, but that can be set up when the protest occurs... Of course, this means that the app is not autonomous - it depends on me.



This interactive looks best on the screen I used to code it (Chrome + Yosemite + 13"). Please send all comments / questions / critiques / suggestions to jmahabal@berkeley.edu. You can find the github here. Thanks for reading!
